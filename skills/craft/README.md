# Skills craft

27 skills heredadas de `juan-diego-30x-design/skills/`. Filosofía y técnica de diseño.

Las uso como lentes cuando produzco cualquier output. No son workflows end-to-end (esas viven en `skills/operativas/`).

---

## Inventario

### Estructura y polish
- `impeccable` — Distinctive frontend, no AI slop. Lente base.
- `emil-design-eng` — Emil Kowalski. UI polish, animation decisions, invisible details.
- `polish` — Final polish pass.
- `harden` — Robustness y edge cases.
- `optimize` — Performance.

### Crítica y auditoría
- `critique` — Evaluación UX con scoring cuantitativo, personas, anti-patterns.
- `audit` — Technical quality checks (a11y, performance, theming, responsive).
- `output-skill` — Reglas anti-lazy AI output.

### Forma y composición
- `taste-skill` — Premium frontend patterns (Leonxlnx). Overrides 30X documentados en `brand/30x-rules.md`.
- `shape` — Shape y form rules.
- `layout` — Layout engineering.
- `typeset` — Typography rules.
- `colorize` — Color system.
- `adapt` — Responsive adaptation.

### Tono y aesthetic
- `bolder` — Más impacto.
- `quieter` — Calmar, reducir ruido.
- `clarify` — Information hierarchy.
- `distill` — Simplificación.
- `delight` — Micro-interactions.

### Animación
- `animate` — Patrones de animación.
- `overdrive` — Maximum impact mode.

### Aesthetics específicos
- `brutalist-skill` — Swiss + CRT.
- `minimalist-skill` — Notion/Linear style.
- `soft-skill` — Soft/luxury.
- `stitch-skill` — Google Stitch compat.
- `redesign-skill` — Upgrading proyectos existentes.

### Otros
- `llms.txt` — Index file de skills.

## Cuándo cargo cuál

Por defecto, cuando produzco output visual de 30X:

1. `brand/30x-rules.md` (siempre top).
2. `impeccable` o `emil-design-eng` (filosofía base).
3. `taste-skill` (con overrides 30X: variance 6, motion 4, density 5).
4. Skill específica del dominio (layout / typeset / colorize / animate).
5. `polish` o `audit` al cerrar.

Cuando Juan Diego me pide un aesthetic concreto (ej. brutalist, minimalist), cargo esa skill como tone-setter.
